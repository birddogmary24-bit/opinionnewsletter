# 세션 로그 생성

오늘 날짜의 커밋 히스토리를 기반으로 세션 로그를 자동 생성합니다.

## 절차

1. 오늘 날짜의 커밋 목록 조회: `git log --oneline --after="어제날짜" --all`
2. 각 커밋의 변경 파일 확인: `git diff --stat <시작커밋>^..<끝커밋>`
3. `docs/` 디렉토리에 `YYYY-MM-DD-session-log.md` 파일 생성

## 로그 포맷

```markdown
# 개발 세션 로그 - YYYY년 M월 D일

## 세션 개요
(전체 작업 요약 1-2줄)

## 작업 내역
### 1. 커밋 제목 (`해시`)
**수정 파일:** 파일 목록
**변경 내용:** 무엇을 왜 변경했는지

## 배포 결과 (배포가 있었다면)
| 항목 | 값 |
|------|-----|

## 이슈 및 해결
| 이슈 | 원인 | 해결 |
|------|------|------|

## 향후 작업 (TODO)
- [ ] 항목

## 변경 파일 요약
(git diff --stat 출력)
```

## 주의사항
- 이미 같은 날짜의 로그가 있으면 덮어쓰지 말고 사용자에게 확인
- 커밋 메시지와 실제 변경 내용을 기반으로 작성 (추측하지 않기)
- 비밀값(비밀번호, API키 등)은 절대 로그에 포함하지 않기
